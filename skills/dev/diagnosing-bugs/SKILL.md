# 系统性诊断 (Diagnosing Bugs)

来源：mattpocock/skills (140k⭐)

对于复杂 bug 和性能回归的纪律性诊断循环。

## 五步循环
1. **复现** — 精确复现问题，确认触发条件
2. **最小化** — 不断缩小范围，找到最小复现用例
3. **假设** — 提出假设，设计实验验证
4. **探测** — 用日志/断点/监控工具验证假设
5. **修复+回归测试** — 修复后确认不引入新问题

## 适用场景
- 难以复现的间歇性 bug
- 性能回归
- 跨模块/跨系统的复杂问题
- 调试多轮仍未解决的老大难
