# Ralph Loop — 快速反馈循环迭代

核心机制：每次完成后将**同一个需求**反馈回来继续迭代，直到**真正完成**。

## 工作机制
1. 接受一个任务/需求
2. 执行并产出结果
3. 完成后不退出，而是**把同一需求再次反馈给你**
4. 你看到上次的成果和 git 历史，继续迭代改进
5. 只有任务被"完全且明确地完成"时才能退出循环

## 适用场景
- 需要反复打磨的 UI 组件
- 算法/逻辑的渐进式优化
- 多轮迭代才能满意的设计稿
- 实验性开发需要频繁调整

## 规则
- **不能为了退出循环而谎称完成**。只有条件完全为真时才能输出完成承诺
- 每次迭代都可以看到前一次的工作成果（文件和 git 历史）
- 循环设计为持续直到真正的完成，不提前退出
