# 架构改善 (Improve Codebase Architecture)

来源：mattpocock/skills (140k⭐)

定期扫描代码库发现架构深化机会，防止AI加速开发导致的代码腐化。

## 工作方式
1. 扫描代码库，发现架构问题（深度不足、模块耦合、命名混乱）
2. 生成可视化 HTML 报告
3. 逐项审查，决定改进顺序
4. 每次改进一个关注点

## 适用频率
- 建议每3-5天运行一次
- 或每次完成一个大型功能后运行

## 检查维度
- 模块是否过深/过浅？
- 接口是否简洁？
- 是否存在紧耦合？
- 是否有重复概念？
- 测试是否覆盖核心路径？
